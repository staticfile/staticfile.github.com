1. Background & Purpose

As Google Ajax Library, Microsoft ASP.net CDN, SAE, Baidu, Upyun and etc are providing free CDN for JS libs, but most of them only provide services for JS libs. We find that CDNJS storing much more libs, JS / CSS / Images / Swf, whereas it is slow in China and DO NOT provide libs that created by Chinese developers.

Thus we decide to provide a warehouse to store open-source libs as much as possible, and serve them via a more stable & fast CDN. In addition, we also provide you an entrance for submitting libs, including JS / CSS / Images & SWF files.

2. How to Sumbit A Lib

Fork the repo，create a folder under libs/, the folder should contain at least one lib. Like：

.
├── jquery                    # ... name of the lib
│   ├── 1.8.3                 # ... version
│   |   └── jquery.min.js     # ... file name, w/o version number
│   └── 1.9.1
│       └── jquery.min.js
└── package.json              # ... for the latest version

Edit package.json, in which contains the information of the LATEST version of the lib. Here's an example file in real-world:libs/typo.css/package.json.

{
  // please keep the folder name the same with the file name
  "name": "name of lib",
  "filename": "core file of the lib, like XYZ.js",
  "version": "1.0",

  // misc
  "description": "description of the lib",
  "homepage": "the homepage of the lib, like http://staticfile.org",
  "keywords": [],
  "maintainers": [
    {
      "name": "author or maintainer",
      "web": "http://staticfile.org",
      "mail": "trimidea@gmail.com"
    }
  ],
  "repositories": [
    {
      // git, svn, hg & etc.
      "type": "git",
      "url": "https://github.com/staticfile/static"
    }
  ]
}

Then, send a Pull Request to us. We will review it as soon as possible. Once the Pull Request is merged, the files of the lib are accessible.

In order to keep fast and stable, we suggest you only submit libs that already have influences -- like, has 20 stars on Github.

3. Updates

The repo fetches updates from CDNJS everyday as well as users' pull requests. If you can't find the right version at staticfile.org. Please open an issue or send us a Pull Request.
